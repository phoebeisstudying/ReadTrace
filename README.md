# 阅迹壁纸 ReadTrace

阅迹壁纸是一个面向文石 BOOX 阅读器的小工具。它可以读取 NeoReader 的本地阅读记录，也可以接入微信读书 API，生成适合墨水屏锁屏/屏保使用的阅读壁纸。

当前主要支持三类壁纸：

- 统计壁纸：把最近阅读、阅读时长、书单、进度、图表和备注排成一张“阅读账单”。
- 当前阅读封面：尝试使用最近阅读书籍的封面生成壁纸。Neo 阅读器来源只读取本地书籍封面；微信读书来源会联网获取并缓存封面。
- 月历封面墙：把一个月里的每日阅读记录排成月历视图，在有阅读记录的日期格中堆叠书籍封面。

项目只通过 GitHub Release 分发，不上架应用商店。

## 适用设备

应用主要为文石 BOOX 设备设计，测试设备为 Leaf 5。只要设备能通过 NeoReader 写入阅读记录，理论上都可以尝试使用。

内置阅读器尺寸预设包括 Poke、P6、Leaf、Note、Tab、T10、T13 等常见型号。首次打开时会尝试根据设备型号自动匹配，匹配不到时默认使用 Leaf 5。

## 安装方式

1. 打开 GitHub Release 页面：
   https://github.com/wberry9813/ReadTrace/releases

2. 下载适合设备的 APK。

   Leaf 5 等近年的 BOOX 设备通常使用：

   ```text
   app-arm64-v8a-release.apk
   ```

   如果安装失败，再尝试：

   ```text
   app-armeabi-v7a-release.apk
   ```

3. 把 APK 复制到阅读器，使用文件管理器打开安装。

4. 如果系统提示“不允许安装未知来源应用”，请按系统提示为文件管理器或浏览器开启安装权限。

5. 安装完成后打开“阅迹壁纸”。

6. 如果 BOOX 系统把应用加入冻结/冷冻列表，请在桌面长按“阅迹壁纸”图标，进入应用设置或优化设置，把应用解冻，并允许后台运行。否则熄屏触发、定时刷新和开机后自动恢复可能不会稳定执行。

## 首次使用

首次打开后，建议按下面顺序操作：

1. 进入“设置”页。
2. 按系统提示授予文件访问、图片访问等权限。
3. 如需使用自定义字体，点击“选择字体目录”，选择设备的 `存储/Fonts` 目录并确认授权。
4. 确认“阅读器尺寸预设”是否为自己的设备型号；如果列表里没有合适尺寸，选择顶部的“自定义分辨率”，填写最终壁纸宽度和高度。
5. 选择“数据来源”和“壁纸类型”。
6. 如果要使用微信读书或混合来源，先在“微信读书”设置组里填写 API Key，并点击测试连接。
7. 点击“刷新预览”，确认 App 能读到阅读记录或书籍封面。
8. 点击“生成壁纸”，把图片保存到设备图片目录。
9. 到文石系统的屏保/壁纸设置里选择生成的图片。

生成的壁纸会覆盖保存为同一张图片，路径通常是：

```text
/storage/emulated/0/Pictures/NeoReader/neoreader_wallpaper.png
```

这样后续自动刷新时，不需要反复重新选择屏保图片。

## 权限说明

应用可能需要以下权限或系统设置：

- 文件和媒体访问：用于读取本地书籍封面、保存生成的壁纸图片，以及写入调试日志。
- 所有文件访问权限：部分 BOOX 系统上，读取本地书籍封面或写入固定图片路径需要开启该权限。
- 字体目录授权：用于读取 `存储/Fonts` 中的 `ttf` / `otf` 字体。这个权限通过系统文件选择器授予，只需要选择 Fonts 文件夹。
- 网络权限：用于检查 GitHub Release 是否有新版本，以及在启用微信读书或混合来源时请求微信读书 API。
- 开机启动：用于设备重启后恢复每日定时或熄屏监听。
- 后台运行/解除冻结：用于保证熄屏触发、内容变化监听和每日定时能执行。
- 前台服务：用于在开启熄屏触发时维持必要的监听服务。

NeoReader 阅读数据来自文石系统的本地 ContentProvider，正常情况下不需要你手动授予单独的“读取阅读记录”权限。如果 App 无法读到数据，请先确认使用的是文石 NeoReader，并尝试打开一本书、退出阅读界面后再刷新预览。

Neo 阅读器来源的封面壁纸只读取本地书籍和本地缓存封面，不会联网下载封面。微信读书来源会联网获取封面，并缓存到 App 私有目录。

## 页面说明

顶部有四个主要入口：

- 设置：调整数据来源、排版、字体、图表、自动刷新等参数。
- 预览：在 App 内查看当前设置生成的效果，不会保存真实壁纸文件。
- 刷新预览：重新读取 NeoReader 数据并更新预览。
- 生成壁纸：把当前预览对应的壁纸保存到图片目录。

预览页只是 App 内部显示；只有点击“生成壁纸”或自动刷新触发时，才会写入真实图片文件。

## 壁纸类型

### 统计壁纸

统计壁纸最稳定，适合长期作为锁屏阅读账单使用。它会根据设置读取阅读记录，显示周期、设备、阅读时长、书籍列表、进度、图表和备注。

不同数据来源下的统计口径：

- Neo 阅读器：来自文石本地阅读统计，适合离线使用。
- 微信读书：来自微信读书 API，书籍进度使用微信读书返回的百分比。
- 混合来源：本地和微信读书时长相加，书单合并排序；如果微信读书读取失败，会降级为本地数据。

### 当前阅读封面

封面模式会尝试读取最近阅读书籍的封面。

Neo 阅读器来源依赖 NeoReader 把当前书籍信息写入元数据，因此有一个常见现象：

必须先退出当前正在阅读的书籍，然后锁屏，才更容易刷新到最新封面。如果在书籍打开状态下直接锁屏，往往仍显示旧封面，通常下一次锁屏才会生效。

微信读书来源会联网读取当前阅读书籍并缓存封面。自动模式下通常在解锁后刷新封面，下一次锁屏显示最新结果。

混合来源会比较本地 Neo 阅读器和微信读书的最近阅读时间，谁更新就优先使用谁的封面；如果该来源封面失败，再回退另一来源。

### 自动封面优先

自动刷新时优先尝试封面壁纸。如果没有拿到可用封面，则回退到统计壁纸。

### 月历封面墙

月历封面墙会生成类似读书月历的壁纸：顶部显示月份，主体为月历网格，有阅读记录的日期会堆叠展示当天相关书籍封面，并在右下角显示当日阅读时长。

Neo 阅读器月历使用本地阅读数据。文石部分固件的阅读统计事件不直接包含书籍路径，因此 App 会结合阅读事件时间和 NeoReader 元数据中的最近访问时间进行近似匹配：

- 每日阅读时长来自本地阅读统计事件。
- 每日封面来自本地书籍封面或 App 已缓存封面。
- 如果统计事件没有书籍路径，日期格里的封面会按时间进行近似归因。
- 生成摘要和日志中会标记 `近似匹配`，便于排查。

微信读书月历完全读取 `WEREAD` 数据库，不会混入 Neo 数据。每日总时长来自微信接口的精确日统计。首次同步时，会把当前书架中每本书的最近阅读时间作为安装前历史，每本书只在最后阅读日补一条封面记录；安装后的记录继续通过快照差分确认。无法确认具体书籍的日期只显示时长，不会使用月度排行猜测封面。

混合月历按日期合并 Neo 与微信读书数据库记录：同一天的总时长相加，同名书籍去重并合并书籍时长，封面堆叠继续遵循“阅读最长 / 阅读最短 / 最近打开”设置。日期右上角使用 `N`、`W`、`M` 标明当天数据来自 Neo、微信读书或两者混合；微信联网失败时使用已有缓存和本地数据。

联网刷新失败时，手动预览仍可查看数据库缓存；自动刷新不会用旧缓存覆盖现有壁纸，而会等待后续重试。

## 数据与统计选项

### 阅读数据落库

“启用阅读数据落库”默认关闭，适合 CPU、存储和电池资源有限的墨水屏设备。

- 关闭：不执行首次历史回填、启动增量维护、解锁快照采集以及 Neo/微信日级数据写入；已有数据库内容会保留，不会被删除。
- 开启：维护 Neo 与微信读书的每日时长、书籍、封面缓存和最近阅读时间，为微信月历及混合月历持续积累数据。
- Neo 统计壁纸、当前封面和 Neo 月历仍可直接读取本地数据，不要求开启落库。
- 微信月历和混合月历依赖日级数据库；关闭时手动预览只能使用之前已保存的缓存，自动任务会保留现有壁纸并跳过远程月历刷新，避免无效重试。新安装设备需要开启后才能逐步建立月历数据。

如果只使用 Neo 统计壁纸或当前封面，建议保持关闭。需要微信月历、混合月历或希望长期保存每日阅读记录时再开启。

### 阅读器尺寸预设

选择目标设备的屏幕尺寸和分辨率。生成图片会按该尺寸输出，预览页会按 App 页面空间等比例缩放显示。

如果列表中有匹配到本机型号的预设，会在选项上做标记。匹配不到时默认 Leaf 5。

列表顶部提供“自定义分辨率”。选择后会显示“自定义宽度(px)”和“自定义高度(px)”两个输入项，适合以下情况：

- 设备不在内置预设列表中。
- 系统壁纸实际尺寸与屏幕物理分辨率不同。
- 需要针对锁屏裁切效果微调输出尺寸。

宽高输入范围为 `300~4000` 像素。请填写最终生成图片的宽和高，例如窄屏竖向壁纸可填写 `824 x 1648`，Leaf 5 预设输出为 `1264 x 1680`。只有选择“自定义分辨率”时，自定义宽高才会生效；选择具体设备预设时仍使用该预设的固定尺寸。

### 统计周期

可选范围：

- 当天：只统计今天。
- 昨天：只统计昨日。
- 本周：按本周范围统计。
- 上周：回看上一周。
- 最近 7 天：滚动统计最近 7 天。
- 最近 30 天：滚动统计最近 30 天。
- 自定义起止：手动选择开始和结束日期。

### 数据来源

- Neo 阅读器：读取文石本地 NeoReader 数据库，优先按阅读时长事件统计。适合离线使用，也是最省电、最稳定的数据来源。
- 微信读书：联网读取微信读书统计、当前阅读书籍和封面。需要在设置页填写 API Key。
- 混合来源：把 Neo 阅读器和微信读书的统计时长相加，书单按阅读时长合并排序，封面按最近阅读来源选择。

混合来源说明：

- 统计壁纸：本地阅读时长 + 微信读书时长；书单按合并后的阅读时长排序。
- 封面壁纸：谁最近阅读就优先使用谁的封面；如果该来源封面获取失败，再回退另一来源。
- 自动刷新：混合来源包含微信读书联网数据，因此不会在熄屏瞬间请求网络。它会在解锁后或每日定时时刷新，成功后覆盖保存图片，通常下一次锁屏看到新图。
- 降级策略：如果微信读书 API、网络或 Key 出错，混合统计会继续使用本地数据；预览/生成摘要和日志会标出微信读取失败。

如果只想完全离线、尽量省电，选择 Neo 阅读器。如果想把微信读书和本地阅读器的阅读时间汇总到同一张账单，选择混合来源。

### 微信读书设置

微信读书相关功能依赖 API Key。配置后可以读取阅读统计、当前阅读书籍、封面和书籍进度。

建议使用顺序：

1. 在设置页找到“微信读书”设置组。
2. 填写 API Key。
3. 点击“测试连接”，确认能读到书架信息。
4. 点击“测试统计”，确认能读到阅读时长。
5. 点击“缓存最近封面”，确认封面可以保存到本机。
6. 将“数据来源”切换为“微信读书”或“混合来源”。
7. 点击“刷新预览”或“生成壁纸”。

微信读书统计周期会跟随上方“统计周期”设置，例如今天、昨天、本周、上周、最近 7 天、最近 30 天和自定义周期。由于微信读书 API 按月读取明细，跨月周期会自动按月份请求后再按日期过滤。

微信读书封面会缓存到 App 私有目录。再次生成同一本书封面时优先使用缓存；如果切换到新的当前阅读书籍，会重新获取并缓存新封面。

App 会在打开时以及微信来源的解锁联网刷新时保存书架和本月排行快照。首次成功同步只建立基线，不把现有累计数据误记到当天；后续检测到书籍最近阅读时间变化时，才写入确认的日级书籍记录。若本月排行仍包含该书，会用两次快照的累计时长差作为本次增量；否则保留“当天读过该书”的记录但时长记为未知。网络失败不会删除已有快照和日级记录。

微信接口只能返回每日总时长、周期书籍排行和每本书当前的最近阅读时间，无法直接还原安装 App 之前“每天具体读了哪些书”。因此微信月历会采用保守策略：每日时长按接口精确显示；安装前历史只把书架中每本书的最近阅读时间作为最后一次出现日期；安装后的日级书籍记录通过快照差分逐步积累，越用越准。

### 时长显示单位

- 小时：显示为更适合阅读的格式，例如 `1小时20分钟`。
- 分钟：全部换算成分钟，适合核对数据。

## 书单筛选

- 最近阅读包含未读：开启后会把 `readingStatus=0` 的记录也纳入最近阅读；关闭后会尽量排除只进过书库但没开始读的书。
- 书单筛选状态：可选择全部、仅在读、仅已读完。
- Top N：控制壁纸最多显示几本书。默认最多 5 本；如果同时显示图表、条码和备注，3 本会更宽松。
- 最小时长阈值：小于该分钟数的阅读事件会被忽略，用于过滤误打开。

## 排版与字体

- 壁纸标题：可编辑右上角标题，例如“阅读账单”。
- 标题字号 / 正文字号：控制整张壁纸的字重和可容纳内容。
- 单号数字模式：可选择月日、随机数或自定义数字。
- 单号数字字号：数字变大时会向上扩展，减少对下一行的挤压。
- 进度行显示：可开关书籍进度和阅读状态。
- 进度显示方式：可选择页数或百分比。
- 作者行显示：开启后会在进度行上方显示作者；如果 NeoReader 元数据没有作者，会显示未知。
- 标题字体 / 正文字体：可选择系统字体或已授权字体目录中的字体。
- 选择字体目录：选择设备的 `存储/Fonts` 目录后，可以使用里面的 `ttf` / `otf` 字体。

## 图表选项

- 显示图表：打开后展示阅读时间分布；如果空间紧张可以关闭。
- 图表样式：折线或柱状。
- 峰值标签：显示最高阅读时长标签。
- Y 轴最大值：自动或固定。固定值适合不同周期之间保持同一比例。

图表横轴规则：

- 当天 / 昨天：按小时。
- 本周 / 上周 / 最近 7 天：按天。
- 最近 30 天：按天。
- 自定义不超过 14 天：按天。
- 自定义 15 到 90 天：按周。
- 自定义超过 90 天：按月。

## 底部备注与条码

- 不显示：底部留白更干净。
- 只显示备注：显示一行自定义文字。
- 条码 + 备注：显示装饰条码和备注。

条码目前偏装饰用途，不保证所有扫码软件都能识别。备注文本会显示在壁纸底部，适合放一句座右铭、读书计划或周期说明。

## 自动刷新

自动刷新会覆盖保存同一张壁纸图片，便于文石屏保继续使用原路径。

可选模式：

- 每日定时一次：省电，适合稳定更新。
- 熄屏触发：更及时，但会增加唤醒次数和耗电。

熄屏触发受“熄屏最小间隔”限制。间隔越短越及时，也越容易增加耗电。NeoReader 常在退出当前书籍或会话结束后才写入最新元数据，所以可能出现本次锁屏仍是旧图、下一次锁屏才生效的情况。

微信读书和混合来源包含联网请求，不会在熄屏瞬间请求网络。自动模式下会在解锁后等待网络恢复并重试；选择每日定时模式时，也会在设定时间执行联网同步。成功后覆盖保存同一张图片，因此常见体验是：解锁或定时刷新成功后，下一次锁屏看到新图。

如果希望自动刷新稳定工作，请在 BOOX 系统中确认“阅迹壁纸”没有被冻结，并允许后台运行。常见入口是桌面长按应用图标，进入应用信息、应用优化或冻结管理，然后关闭冻结限制。不同固件名称可能略有差异。

## 版本与更新

设置页底部有“版本与更新”区域：

- 检查更新：访问 GitHub Release，比较本地版本和最新版本。
- 打开 Release 页面：跳转到 GitHub Release 页面。

应用只检查并跳转，不会自动下载或安装 APK。

如果 GitHub 上的版本和本地版本一致，会显示类似：

```text
已是最新（当前 v1.0.0）
```

## 常见问题

### 检查更新失败

请确认设备能访问 GitHub。应用会依次尝试 GitHub Release API、Release 列表、Tag 列表和 Release 页面跳转地址。如果仍然失败，可以导出日志排查。

### 熄屏触发或定时刷新不生效

先确认应用没有被冻结。BOOX 系统可能会为了省电冻结不常打开的应用，需要在桌面长按“阅迹壁纸”图标，进入应用设置或优化设置，解除冻结并允许后台运行。然后重新打开 App，确认“自动刷新”已开启。

### 封面壁纸没有更新

先退出当前正在阅读的书籍，再锁屏。NeoReader 的当前书籍信息通常要在退出阅读会话后才会写入数据库。如果第一次锁屏仍然是旧封面，通常下一次锁屏会生效。

如果数据来源是微信读书或混合来源，封面刷新依赖解锁后的联网预热或每日定时联网任务。请先解锁设备等待几秒，再锁屏查看；如果网络刚恢复，可能需要下一次锁屏才看到新图。

### 混合来源只显示本地数据

混合来源会优先合并本地和微信读书数据。如果微信读书 API Key 无效、网络未恢复、接口失败或 GitHub Skill 服务不可用，App 会降级使用本地数据，避免整张壁纸生成失败。

遇到这种情况可以检查：

- 微信读书 API Key 是否已填写。
- 设置页“测试连接”是否成功。
- 设备当前是否能联网。
- 日志中的 `Mixed stats`、`WeRead wallpaper stats failed`、`weReadFailures` 等字段。

### 作者显示未知

作者来自 NeoReader 元数据。如果书籍本身没有作者信息，或者 NeoReader 没有解析出来，就会显示未知。可以关闭作者行节省空间。

### 字体列表里没有 Fonts 目录中的字体

在“选择字体目录”里选择设备的 `存储/Fonts` 目录，不是单个字体文件。授权后回到设置页，标题字体和正文字体列表会出现该目录下的字体。

### 壁纸内容显示不全

减少 Top N，关闭图表或底部条码，或者调小标题/正文字号。统计壁纸会尽量自适应，但墨水屏竖版空间有限。

## 调试日志

如果遇到问题，可以在设备上导出以下日志并反馈：

```text
/storage/emulated/0/Download/neoreader_debug_log.txt
/storage/emulated/0/Download/neoreader_auto_refresh_log.txt
```

日志中会记录元数据字段、封面命中来源、自动刷新触发原因、更新检查 HTTP 状态等信息。

## 开发构建

本项目当前核心功能在 Android 原生 Kotlin 代码中。

Debug 构建：

```sh
cd android
JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home" \
GRADLE_USER_HOME="/Users/dmer/Dev/NeoReaderReaderRecords/.gradle-home" \
./gradlew assembleDebug --no-daemon
```

Release 构建：

```sh
cd android
JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home" \
GRADLE_USER_HOME="/Users/dmer/Dev/NeoReaderReaderRecords/.gradle-home" \
./gradlew assembleRelease --no-daemon
```

Release APK 输出位置：

```text
android/app/build/outputs/apk/release/
```
